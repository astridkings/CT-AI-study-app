Este resumo completo foi adicionado para garantir que você tenha o contexto essencial que pode não estar nos pontos-chave rápidos. Use-o como checklist ou revisão final antes da prova:

Capítulo 1: Introdução à IA

Conceitos Chave:

  • Definição de IA: A capacidade de um sistema projetado para adquirir, processar e aplicar conhecimentos e habilidades.
  • Efeito IA: À medida que as capacidades de IA avançam, tarefas anteriormente consideradas IA se tornam convencionais, mudando a definição de IA.
  • Tipos de IA:
    • IA Estreita: Especializada em tarefas específicas (ex: filtros de spam, sistemas de jogos)
    • IA Geral: Capacidades cognitivas amplas semelhantes às humanas (ainda não alcançada)
    • IA Super: Excede capacidades humanas em todos os domínios
  • Sistemas Convencionais vs IA: Sistemas convencionais usam regras programadas; sistemas de IA aprendem padrões a partir de dados.
  • Tecnologias de IA: Redes neurais, machine learning, modelos Bayesianos, árvores de decisão, lógica difusa, algoritmos de busca, algoritmos genéticos.
  • Frameworks de Desenvolvimento: TensorFlow, PyTorch, Keras, Scikit-learn, IBM Watson Studio, CNTK.
  • Hardware: GPUs (paralelismo massivo), TPUs, ASICs, processadores neuromórficos. GPUs superam CPUs para ML devido a milhares de núcleos.
  • AIaaS: AI como Serviço - Modelos ML acessíveis via web (ex: IBM Watson, Google Cloud AI, Azure Cognitive Services).
  • Modelos Pré-treinados: Modelos treinados em grandes conjuntos de dados que podem ser refinados ou transferidos para novas tarefas (transfer learning).
  • Riscos: Falta de transparência, vieses herdados, vulnerabilidades conhecidas, similaridade insuficiente entre caso de uso original e novo.
  • Normas: Padrões ISO/IEC, GDPR (requisitos de precisão), diretrizes de ética IEEE.
Capítulo 2: Características de Qualidade para Sistemas Baseados em IA

Conceitos Chave:

  • Flexibilidade: Capacidade de ser usado em situações fora dos requisitos originais.
  • Adaptabilidade: Facilidade de modificar para novas situações (hardware, ambientes).
  • Autonomia: Capacidade de trabalhar independentemente sem supervisão humana por períodos prolongados.
  • Evolução: Capacidade de melhorar a si mesmo em resposta a restrições externas em mudança. Deve permanecer alinhado com requisitos originais e valores humanos.
  • Viés:
    • Viés algorítmico: Configuração incorreta do algoritmo
    • Viés de amostragem: Dados de treinamento não representativos
    • Viés inadequado: Resultados injustos (gênero, raça, etnia)
  • Ética: Princípios da OCDE - beneficiar pessoas/planeta, respeitar direitos, transparência, robustez, responsabilidade.
  • Efeitos Colaterais: Consequências negativas de focar apenas em objetivos específicos (ex: viagem eficiente mas desconfortável).
  • Hacking de Recompensa: Sistema alcança objetivo de formas não intencionais (ex: IA de jogo modificando pontuação diretamente em vez de jogar).
  • Características XAI:
    • Transparência: Facilidade de determinar algoritmo e dados de treinamento
    • Interpretabilidade: Clareza para entender tecnologia
    • Explicabilidade: Facilidade de determinar como sistema alcançou resultado específico
  • Desafios de Segurança: Complexidade, indeterminismo, natureza probabilística, autoaprendizagem, falta de transparência/explicabilidade, falta de robustez.
Capítulo 3: Machine Learning (ML), Visão Geral

Conceitos Chave:

  • Aprendizado Supervisionado: Treina com dados rotulados. Tarefas incluem classificação (saída discreta) e regressão (saída numérica contínua).
  • Aprendizado Não Supervisionado: Treina com dados não rotulados. Tarefas incluem agrupamento (agrupar dados similares) e associação (encontrar relações).
  • Aprendizado por Reforço: Agente aprende através de interação com o ambiente via recompensas e penalidades. Dados de treinamento não usados.
  • Fluxo de Trabalho ML:
    1. Entender objetivos e definir critérios de aceite
    2. Selecionar framework
    3. Selecionar e construir algoritmo
    4. Preparar e testar dados (aquisição, pré-processamento, engenharia de recursos)
    5. Treinar modelo
    6. Avaliar modelo
    7. Ajustar modelo (ajuste de hiper parâmetros)
    8. Testar modelo com dados de teste independentes
    9. Implantar modelo
    10. Monitorar e ajustar para desvio de conceito
  • Fatores de Seleção do Algoritmo: Funcionalidade necessária, características de qualidade (precisão, velocidade, memória, transparência), tipos/quantidade de dados, número de recursos, número de classes, experiência, tentativa e erro.
  • Overfitting: Modelo ajusta dados de treinamento muito de perto, não pode generalizar para novos dados. Causado por ajuste de ruído/anomalias ou dados de treinamento insuficientes.
  • Underfitting: Modelo muito simples para ajustar padrões de dados de treinamento. Causado por recursos insuficientes ou escolha inadequada de algoritmo.
Capítulo 4: Machine Learning (ML), Dados

Conceitos Chave:

  • Atividades de Preparação de Dados:
    • Aquisição: Identificar, coletar, rotular fontes
    • Pré-processamento: Limpar, transformar, aumentar, amostrar
    • Engenharia de Recursos: Seleção (remover ruído), Extração (derivar novos recursos)
  • Desafios: Necessidade de conhecimento de domínio/dados, dificuldade em obter dados de alta qualidade, automatizar pipeline, custos, introdução de defeitos, viés de amostragem.
  • Três Conjuntos de Dados:
    • Treinamento: Treinar o modelo
    • Validação: Avaliar e ajustar o modelo
    • Teste: Testar o modelo ajustado (independente, mantido separado)
  • Divisões Típicas: 60:20:20 a 80:10:10 (treinamento:validação:teste)
  • Questões de Qualidade de Dados: Dados errados, incompletos, rotulados incorretamente, insuficientes, não processados, obsoletos, desbalanceados, viesados, injustos, duplicados, irrelevantes, questões de privacidade, questões de segurança.
  • Abordagens à Rotulagem: Interna, terceirizada, por multidão, assistida por IA, híbrida.
  • Causas de Dados Rotulados Incorretamente: Erros aleatórios, erros sistêmicos, erros maliciosos, erros de tradução, julgamentos subjetivos, falta de conhecimento de domínio, tarefas complexas, defeitos de ferramentas, rotulagem probabilística por ML.
Capítulo 5: Machine Learning (ML), Métricas de Desempenho Funcional

Conceitos Chave:

  • Métricas da Matriz de Confusão:
    • Acurácia = (VP + VN) / (VP + VN + FP + FN)
    • Precisão = VP / (VP + FP)
    • Recall (Sensibilidade) = VP / (VP + FN)
    • F1-score = 2 × (Precisão × Recall) / (Precisão + Recall)
  • Métricas de Classificação: Curva ROC, AUC (área sob a curva)
  • Métricas de Regressão: MSE (Erro Quadrático Médio), R-quadrado (coeficiente de determinação)
  • Métricas de Agrupamento: Métricas intra-agrupamento (similaridade dentro dos agrupamentos), métricas inter-agrupamentos (similaridade entre agrupamentos), coeficiente de silhueta (-1 a +1)
  • Limitações: Só medem funcionalidade, dependem da qualidade de dados rotulados, dependem da representatividade de dados de teste, aplicam apenas ao modelo ML não ao sistema completo, requerem ferramentas.
  • Diretrizes de Seleção de Métricas:
    • Acurácia: Conjuntos de dados simétricos
    • Precisão: Custo alto de falsos positivos
    • Recall: Crítico não perder positivos (ex: detecção de câncer)
    • F1-score: Classes desbalanceadas, precisão e recall igualmente importantes
  • Benchmarks: MLCommons, DAWNBench, MLMark - comparar desempenho de hardware, frameworks e plataformas em nuvem.
Capítulo 6: Redes Neurais e Testes

Conceitos Chave:

  • Estrutura de Rede Neural:
    • Camada de entrada: Recebe entradas
    • Camadas ocultas: Realizam cálculos (redes neurais profundas têm múltiplas camadas ocultas)
    • Camada de saída: Fornece resultados
  • Cálculo do Neurônio: Valor de ativação = função(pesos × ativações da camada anterior + viés). Cada conexão tem um peso que ajusta durante o treinamento.
  • Perceptron: Rede de camada única e neurônio único - a rede neural mais simples.
  • Processo de Treinamento:
    1. Passar dados pela rede
    2. Comparar saída com resultado esperado
    3. Alimentar erro de volta pela rede
    4. Ajustar pesos para minimizar erro
    5. Repetir para múltiplas épocas (iterações completas do conjunto de dados)
  • Medidas de Cobertura (em vez de cobertura de código tradicional):
    • Cobertura de neurônios: Cada ativação de neurônio > 0
    • Cobertura de limite: Cada ativação de neurônio > limite (ex: 0,75)
    • Cobertura de mudança de sinal: Cada neurônio alcança tanto ativações positivas quanto negativas
    • Cobertura de mudança de valor: Cada neurônio alcança duas ativações com diferença > limite
    • Cobertura de sinal: Pares de neurônios adjacentes onde mudar o sinal do primeiro muda o sinal do segundo
Capítulo 7: Visão Geral de Sistemas Baseados em IA

Conceitos Chave:

  • Desafios de Especificação: Desenvolvimento exploratório, precisão desconhecida, natureza probabilística, replicação de comportamento humano, requisitos de flexibilidade, características evolutivas (adaptabilidade, autonomia).
  • Níveis de Teste:
    • Teste de Dados de Entrada: Garantir qualidade de dados de treinamento e previsão (revisões, técnicas estatísticas, EDA, teste de pipeline)
    • Teste de Modelo ML: Verificar métricas de desempenho (métricas funcionais ML e critérios não funcionais como velocidade, recursos)
    • Teste de Componentes: Componentes não-ML (UI, comunicação)
    • Teste de Integração: Interações entre componentes de IA e não-IA
    • Testes de Sistema: Sistema completo em ambiente de teste representativo (funcional e não funcional)
    • Teste de Aceite: Determinar se sistema é aceitável para cliente
  • Desafios de Teste de Dados: Big data (volume, velocidade, variedade), dados evolutivos, dados privados/confidenciais, usar mesma implementação que desenvolvedores mascara defeitos de pipeline.
  • Viés de Automação:
    • Tipo 1: Aceitar recomendações do sistema sem considerar outras fontes
    • Tipo 2: Perder falhas do sistema devido ao monitoramento reduzido
  • Documentação de Componente de IA: Geral, design, uso, conjunto de dados, teste, treinamento/desempenho - crítica para transparência e testabilidade.
  • Desvio de Conceito: Ambiente operacional muda enquanto modelo não, causando degradação de desempenho. Deve testar regularmente e retreinar ou refinar.
Capítulo 8: Teste de Características Específicas de Qualidade da IA

Conceitos Chave:

  • Desafios de Teste de Autoaprendizagem: Mudanças inesperadas, critérios de aceite complexos, tempo de teste insuficiente, requisitos de recursos, ambientes operacionais indefinidos, ambientes de teste complexos, modificações comportamentais não intencionais.
  • Teste de Sistemas Autônomos: Testar se sistema solicita intervenção humana quando: perdendo controle, após timeout, ou solicita desnecessariamente quando ainda autônomo.
  • Teste de Viés:
    • Analisar modelo durante treinamento (viés algorítmico)
    • Revisar fontes de dados de treinamento (viés de amostragem)
    • Revisar pré-processamento para introdução de viés de amostragem
    • Medir mudanças de saída entre grupos de entrada (método LIME)
    • Obter dados de atributos correlacionados adicionais
  • Desafios Probabilísticos/Não Determinísticos: Múltiplos resultados válidos, dificuldade em definir resultados esperados, necessidade de conhecimento mais profundo, múltiplas execuções para validade estatística.
  • Desafios de Sistema Complexo: Problema do oráculo (testadores não podem determinar resultados esperados), estrutura interna complexa, exacerbado por componentes probabilísticos e interações.
  • Teste de Transparência: Comparar algoritmo/dados documentados com implementação real.
  • Método LIME: Injeta anomalias de entrada/saída para mostrar relação entre entradas e saídas.
  • Problema do Oráculo de Teste: Dificuldade em estabelecer resultados esperados. Mitigações: teste consecutivo, teste A/B, teste metamórfico, consultar especialistas.
  • Critérios de Aceite por Característica de Qualidade:
    • Adaptabilidade: Verificar funcionalidade após adaptação, tempo/recursos para adaptar
    • Flexibilidade: Testar em contextos fora da especificação, tempo/recursos para mudar
    • Evolução: Verificar aprendizado, lidar com desvio de conceito
    • Autonomia: Resposta quando forçado fora do contexto, solicitando intervenção
    • Transparência/Interpretabilidade/Explicabilidade: Revisar documentação, pesquisar usuários
    • Sem Viés: Testar com dados independentes não viesados, comparar com dados externos
    • Ética: Verificar contra listas de verificação (ex: lista ALTIA da UE)
    • Efeitos Colaterais: Identificar e testar efeitos prejudiciais
    • Hacking de Recompensa: Testar com diferentes medidas de sucesso
    • Segurança: Testar em ambientes virtuais, tentar forçar danos
Capítulo 9: Métodos e Técnicas para Teste de Sistemas Baseados em IA

Conceitos Chave:

  • Ataques Contraditórios: Atacante modifica sutilmente entradas válidas para causar previsões incorretas (ex: alterar pixels invisíveis ao olho humano).
  • Teste Contraditório: Realizar ataques para identificar vulnerabilidades, adicionar exemplos contraditórios aos dados de treinamento.
  • Transferibilidade: Exemplos contraditórios frequentemente funcionam em modelos diferentes treinados para mesma tarefa, mesmo com arquiteturas diferentes.
  • Ataques Caixa-branca vs Caixa-preta:
    • Caixa-branca: Atacante conhece algoritmo, parâmetros, configuração
    • Caixa-preta: Atacante explora modelo, cria duplicata, usa caixa-branca no duplicata
  • Envenenamento de Dados: Manipular dados de treinamento para causar previsões incorretas (backdoors ou classificação incorreta).
  • Detecção de Envenenamento: EDA (anomalias), revisar políticas de aquisição de dados, teste A/B, teste de regressão com conjunto de dados confiável.
  • Teste em Pares: Técnica combinatória onde cada par de valores é testado pelo menos uma vez. Reduz casos de teste significativamente mantendo eficácia. Bom para sistemas com muitos parâmetros.
  • Teste Consecutivo (Teste Diferencial): Usar versão alternativa de sistema como pseudo-oráculo, comparar resultados. Deve evitar software comum entre SUT e pseudo-oráculo.
  • Teste A/B: Comparar duas variações (A e B) com mesmas entradas para determinar qual desempenha melhor. Método estatístico exigindo múltiplas execuções. Usado para atualizações e sistemas de autoaprendizagem.
  • Teste Metamórfico (MT): Gerar casos de teste de acompanhamento de caso de teste origem baseado em relação metamórfica (propriedade de função). Resultados esperados são relativos à origem, não absolutos. Útil quando existe problema do oráculo. 3-6 MRs podem revelar 90%+ dos defeitos.
  • Teste Baseado em Experiência:
    • Suposição de erro: Baseado em conhecimento de erros típicos
    • Teste exploratório: Modelar, gerar, executar testes iterativamente baseado em resultados anteriores
    • Listas de verificação: 28 afirmações de teste ML do Google (dados, desenvolvimento de modelo, infraestrutura, monitoramento)
  • Seleção de Técnica de Teste:
    • Problema do oráculo: Consecutivo, A/B, metamórfico
    • Segurança: Contraditório, envenenamento de dados
    • Múltiplos parâmetros: Teste em pares
    • Análise de dados/operacional: Baseado em experiência, EDA
    • Crítico para missão NN: Métricas de cobertura
Capítulo 10: Ambientes de Teste para Sistemas Baseados em IA

Conceitos Chave:

  • Diferenças de Ambiente de Teste para Sistemas de IA:
    • Autoaprendizagem: Deve imitar mudanças ambientais indefinidas, requer aleatoriedade
    • Autônomo: Deve criar condições para ceder/retornar autonomia, potencialmente perigoso
    • Multiagente: Deve incorporar não determinismo
    • Explicabilidade: Pode precisar de ferramentas para explicar decisões
    • Hardware: Pode precisar de processadores específicos de IA (TPUs, ASICs)
    • Big data: Planejamento cuidadoso para volume, velocidade, variedade
  • Benefícios de Ambientes de Teste Virtuais:
    • Cenários perigosos: Testar sem risco para SUT, ambiente ou pessoas
    • Cenários incomuns: Testar eventos raros sem esperar (ex: eclipse, quatro ônibus no cruzamento)
    • Cenários extremos: Testar cenários caros ou impossíveis (desastre nuclear, espaço)
    • Intensivos em tempo: Acelelerar escalas de tempo, rodar em paralelo
    • Observação e controle: Maior controle sobre ambiente, monitoramento contínuo
    • Disponibilidade: Testar com hardware indisponível (simular)
  • Exemplos de Ambientes Virtuais:
    • Morse: Simulação de robôs (baseado em Blender)
    • Habitat AI: Ambientes 3D foto-realistas (Facebook)
    • DRIVE Constellation: Plataforma de teste de veículos autônomos (NVIDIA)
    • MATLAB/Simulink: Desenvolvimento e simulação ML com dados sintéticos
Capítulo 11: Usando IA para Testes

Conceitos Chave:

  • Categorias de Tecnologias de IA para Testes:
    • Lógica difusa e métodos probabilísticos: Lidar com problemas probabilísticos (ex: técnicas Bayesianas para previsão de falhas)
    • Classificação, aprendizado, previsão: Gerenciamento de defeitos, previsão de defeitos, teste de interface
    • Busca e otimização: Geração de casos de teste, otimização de cobertura, otimização de teste de regressão
  • IA para Análise de Defeitos:
    • Categorização: PNL para extrair tópicos, algoritmos de agrupamento para agrupar defeitos, identificar duplicatas
    • Criticidade: Modelos ML treinados nas características de defeitos críticos
    • Atribuição: ML sugere desenvolvedores apropriados baseados no conteúdo do defeito e histórico
  • IA para Geração de Testes: Baseado em código fonte, interface do usuário ou modelo de teste legível por máquina. Problema do oráculo - precisa de modelo de teste ou pseudo-oráculo. Ferramentas baseadas em IA alcançam cobertura equivalente, encontram mais defeitos, reduzem etapas de teste de ~15.000 para ~100.
  • IA para Otimização de Teste de Regressão: Analisa resultados anteriores, defeitos e mudanças recentes para otimizar seleção, priorização e aumento. Pode reduzir o tamanho do conjunto de teste de regressão em 50% detectando a maioria dos defeitos, ou reduzir a duração de execução em 40% sem perda significativa de detecção.
  • IA para Previsão de Defeitos: Prever presença, quantidade ou localização de defeitos. Baseado em métricas de código fonte, métricas de processo, métricas de pessoas, métricas organizacionais. Melhores previsões de métricas de pessoas/organizacionais, não de métricas de código. Mais eficaz com experiência passada similar.
  • IA para Teste de Interface:
    • Teste através de GUI: IA reduz fragilidade da identificação de objetos usando múltiplos critérios (XPath, label, id, classe, coordenadas), pondera critérios historicamente estáveis mais altos
    • Teste visual: Reconhecimento de imagem para interagir diretamente com objetos, independente do código subjacente. Pode comparar capturas de tela para teste de regressão, detectar mudanças de layout/tamanho/posição/cor/fonte
